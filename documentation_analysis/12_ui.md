# 12. Analisis Antarmuka Pengguna (UI/UX Analysis)

Dokumen ini membedah komponen dan struktur hirarki antarmuka aplikasi GLOW. Sistem menggunakan model Single Page Application (SPA), dengan *Vanilla HTML5, CSS3, dan JavaScript*. Seluruh logika perutean dikoordinasikan dalam fail `client/js/app.js` menggunakan interupsi *Hash Routing* berbasis komponen dinamis.

## 1. Arsitektur Komponen Visual (Design System)
Semua warna dan *font* terpusat pada properti variabel dasar (`client/css/main.css`).

- **Tema Warna:** Variabel nuansa keemasan (`--primary: #d97706`), warna teks malam (`--gray-900: #111827`), dan krem (`--surface: #f9fafb`). Warna merepresentasikan kesan mewah alam dan keramahtamahan lokal (menyerupai aplikasi penginapan butik berkelas).
- **Tipografi Dasar:** Jenis huruf utama modern `Inter` untuk fungsionalitas, disandingkan font serif pilar `Playfair Display` untuk logo dan *heading* Hero untuk memberikan sensasi klasik (elegan).
- **Mobile First Approach:** Seluruh halaman dilengkapi pilar navigasi *Bottom Nav Bar* melayang dan susunan CSS yang secara responsif menyembunyikan navigasi lengkap *Navbar Header* ke dalam modul Sidebar (*Drawer Menu*) pada resolusi kecil.
- **Glassmorphism:** Banyak *card* dan *modal* yang memiliki efek kabur-akrilik (CSS: `backdrop-filter: blur(10px)`) agar kontras dengan foto Gunung Kidul.

## 2. Struktur Modul & Fungsi Halaman

GLOW memiliki antarmuka utama yang termodulasi secara JS *Components*.

### A. Komponen Global (Melekat di setiap Halaman)
1. **Navigation Bar (Header):** Merender menu dinamis berdasarkan deteksi status *Role*. Menyajikan indikator angka lencana (Badge) "Favorit".
2. **Bottom Navigation (Mobile):** Render menu layang di kaki layar peramban gawai (Home, Package, Productivity, Profile).
3. **Auth Modal (Overlay):** Blok pendaftaran/masuk (Form Input Nama, Email, Sandi) yang memunculkan kotak modifikasi (Live Validation merah) jika pola *regex* terdeteksi tidak wajar. Form berubah warna menjadi Hijau cerah untuk mengizinkan (Unlock) Submit Form.

### B. Daftar Halaman Inti (Pages)

| Rute URL Internal | Modul Pemroses | Fungsi Tujuan Layar & Tampilan DOM |
|---|---|---|
| `#search` | `renderSearchPage()` | **Beranda / Pencarian:** Memuat Slideshow Hero dinamis, Filter Pill geser (Kategori Penginapan/Wisata), Slider Bar Harga, Daftar Card Inventaris (Foto, Harga, Ikon Wi-Fi). |
| `#location-detail?id=X`| `renderLocationDetail()` | **Pusat Informasi:** Gambar hero layar penuh, spesifikasi kolom fasilitas (Power Outlet/AC), integrasi bingkai lokasi peta geografis, daftar evaluasi review 4 kriteria, serta tombol Aksi (Favorite & Tambahkan ke Paket). |
| `#package` | `renderPackagePage()` | **Pembuat Paket Custom (Wadah):** Terdiri dari 2 Tab (Buat Sendiri & Rekomendasi AI). Menyajikan daftar kartu mini keranjang belanja yang siap dipesan. Ada rincian nominal penambahan Harga Akhir Kalkulator di bawah kolom. |
| `#booking` | `renderBookingPage()` | **Wizard Konfirmasi Bayar:** Modul ini membedah layar dengan tahapan linear bersyarat: Rincian Paket -> Modul Pemohon (Profil KTP) -> Tanggal Reservasi CheckIn -> Struk Pembayaran Sukses. |
| `#productivity` | `renderProductivityPage()` | **Alat Bekerja Remote:** Jam putar lingkar *Timer Pomodoro* (25 menit). Ada tabel agenda waktu jam dan kolom Input Skala 5 Bintang Kesejahteraan Emosional pengguna harian (Mood). |
| `#favorites` | `renderFavoritesPage()` | **Koleksi Disukai:** Lemari kartu statis berisi direktori properti yang ditandai Hati. Opsi un-favorite secara kolektif. |
| `#review` | `renderReviewPage()` | **Evaluasi Lokasi:** Form Bintang *Rating* kuadran 4 aspek, plus bidang area input opini Teks (opsional). |

### C. Halaman Khusus Peran/Role (Role-Based Dashboards)
1. **Dasbor Pelanggan (`#dashboard-user`):** Profil KTP/Biografi pengguna, Tabel sejarah reservasi (dengan format status kapsul warna-warni PENDING/COMPLETED), dan Chart (Grafik) riwayat *Mood* perjalanan lampau.
2. **Dasbor Pemilik (`#dashboard-owner`):** Halaman ringkasan kartu (*Card Layout*) Total Pengunjung dan Rupiah (*Revenue*). Terdapat modal untuk mengelola CRUD menambah properti penginapan/kafe baru yang memerlukan *input attachment files* (Unggah Berkas Jpg).
3. **Dasbor Admin (`#dashboard-admin`):** Layar tabel masif pengelolaan tingkat server. Opsi tab: Spionase Pengguna, Verifikasi Mitra Bisnis Pending, Matikan status Terbitnya Lokasi (*Toggle IsPublished*). Fitur perlindungan *Alert* JavaScript (Window Prompt) sebelum melakukan tindakan penindasan aset *Hapus Berkas Permanen*.

## 3. Navigasi & Output (UX Flow)
Dalam navigasinya, arsitek UX menempatkan umpan balik (*feedback*) taktil sekecil apapun di sisi dom:
- Interupsi gagal pada proses logika API akan digambar menggunakan *Toast UI Component* (Pesan Notifikasi Latar Hitam melayang selama 3 Detik).
- Gambar-gambar profil & tempat ditambahkan properti `object-fit: cover` dengan penangguhan *Skeleton-loading* CSS agar layar tidak bertumpuk selama koneksi internet jelek (pengalaman respons cepat).
- Transisi halaman (SPA Render DOM) dikondisikan tanpa muat ulang.
